# Firebase Integration Guide for CCC

This document covers how the CCC (Control Center Console) interacts with the Firebase ecosystem, particularly since the client station applications fully migrated from Supabase to Firebase.

## Role of CCC
The CCC backend provides APIs for device approval, status validation, and device management.
All data related to the "Hardware Station Kiosks" is stored in Firestore documents inside the `kiosks` collection.

The Front-End and Station applications interact directly with Firebase Authentication and Firestore to retrieve these approved keys.

## Required Firebase Configuration
If deploying or editing CCC, you must ensure the underlying `maainhome-ccc` Firebase project matches the following security postures:

### 1. Firestore Rules
The backend functions (Go running on Vercel) bypass Firestore rules because they authenticate using Admin/Service Accounts or REST APIs depending on the setup. However, the client applications which connect to this same environment require the following rules:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /profiles/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /pairing_tokens/{document=**} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

### 2. Authentication Domains
Because the client application authenticates directly with Firebase, the domains serving the client app and CCC must be authorized in the Firebase Console:
- Navigate to **Authentication** -> **Settings** -> **Authorized Domains**.
- Ensure both the `maahome.in` and `maainhome-ccc.vercel.app` domains are added.

## Audit Collections
- `kiosks`: Contains all the UUIDs and hardware pins mapped to their approval keys. The CCC Go backend writes here.
- `profiles`: Contains user accounts. The Client Typescript frontend writes here.
- `pairing_tokens`: Contains WebRTC linkage generated by the Client Typescript frontend.
